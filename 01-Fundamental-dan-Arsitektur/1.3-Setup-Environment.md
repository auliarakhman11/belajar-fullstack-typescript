# 1.3. Setup Environment: PostgreSQL, Node.js, dan Repositori Git

Sebelum kita mulai menulis kode untuk Backend dan Frontend, kita harus memastikan sistem di laptop Anda sudah siap dan memiliki dependensi yang dibutuhkan. Karena kita menggunakan arsitektur modern (Next.js dan Express), JavaScript/TypeScript adalah inti dari semuanya.

## 1. Node.js & Package Manager

Node.js adalah mesin _runtime_ yang memungkinkan kita menjalankan JavaScript di luar browser (di server/laptop Anda).

- **Instalasi:** Pastikan Anda memiliki Node.js versi **LTS (Long Term Support)**. Anda bisa mengunduhnya dari situs resmi Node.js.
- **Pengecekan:** Buka Terminal di VS Code (tekan `` Ctrl + ` ``) dan jalankan:
  `node -v`
  `npm -v`
- **Teori Singkat:** `npm` (Node Package Manager) adalah alat yang akan kita gunakan untuk mengunduh seluruh _library_ pendukung, seperti Prisma ORM, Express, dan Tailwind CSS.

## 2. Database PostgreSQL Lokal

Anda telah memutuskan menggunakan PostgreSQL, dan ini adalah pilihan brilian. PostgreSQL adalah RDBMS (Relational Database Management System) kelas _Enterprise_ yang sangat ketat terhadap integritas data. Ini jauh lebih tangguh untuk sistem transaksi keuangan dan relasi multi-outlet.

- **Instalasi:** Jika Anda belum menginstalnya, unduh _installer_ dari situs resmi (EnterpriseDB).
- **Kredensial Penting:** Saat proses instalasi, Anda akan diminta membuat _password_ untuk _superuser_ default yang bernama `postgres`. **Catat password ini baik-baik!** Kita akan sangat membutuhkannya saat menghubungkan Backend (Prisma) ke database di Bab 2.
- **Port Default:** Secara standar, PostgreSQL berjalan di port `5432`.
- **Rekomendasi Alat Visual (GUI):** Untuk mempermudah melihat isi tabel database tanpa harus mengetik _query_ di terminal hitam, sangat disarankan untuk menginstal **DBeaver** atau **pgAdmin**.

## 3. Inisialisasi Workspace (Folder & Git)

Karena sistem kita terbagi dua (API Backend dan UI Frontend), kita akan menyiapkannya berdampingan dalam satu wadah folder utama (`POS-Masterclass`).

Lakukan langkah-langkah praktis ini di terminal VS Code Anda:

1. Pastikan terminal berada di lokasi folder utama Anda:
   `cd jalur/ke/folder/POS-Masterclass`

2. Inisialisasi pelacakan versi proyek dengan Git:
   `git init`

3. Buat dua folder terpisah untuk wadah aplikasi kita nanti:
   `mkdir pos-backend`
   `mkdir pos-frontend`

4. Buat file `.gitignore` di folder root (`POS-Masterclass`). File ini berfungsi untuk memberi tahu Git agar tidak menyimpan file-file berat atau rahasia ke repositori. Isi file `.gitignore` tersebut dengan teks berikut:
   ```text
   node_modules/
   .env
   .DS_Store
   ```
